# portfolio
データ分析レポートのポートフォリオ掲載用のフォルダ
⚾️ 野球トラッキングデータを用いた投球結果（三振）の要因分析とアクション提案

📑 エグゼクティブ・サマリー（要約）

本プロジェクトは、MLB等の野球トラッキングデータを用いて「三振（Strikeout）」を予測するだけでなく、「現場の投手が三振を奪うための具体的なアクション要因」を特定することを目的としています。

単なる「三振予測モデル」は、ブラックボックス化しやすく現場でのアクションに直結しないという課題がありました。そこで本分析では、「ストライク予測（Stage 1）」と「三振予測（Stage 2）」の2段階（Two-Stage）機械学習パイプラインを独自に設計しました。結果として、高い予測精度（LogLoss）を達成しつつ、SHAPを用いた要因分析により「ストライク判定をもらいやすい球質の閾値」という現場へ提案可能なインサイトを抽出しました。

さらに、機械学習を用いたWhat-if分析と統計的仮説検定（対応のあるt検定）により、アクションの投資対効果（ROI）を定量的に証明。現職でのBIツールを用いた経営レポート構築経験を活かし、本分析結果を「現場のコーチ・選手向けのダッシュボード」として可視化・運用するまでのプロセスを提示します。

1. 🎯 ビジネス課題とアプローチ

課題の再定義

「三振を予測せよ」という目的変数に対し、直接モデリングを行うだけでは「なぜ三振が取れるのか」という文脈（見逃しなのか、空振りなのか、追い込んでいるからか）が欠落します。現場が求めているのは「AIの予測確率」ではなく、「明日からどう投げればよいか」です。

アプローチ（Two-Stageモデルの採用）

「三振を奪うには、まずストライク（判定またはスイング）を取る必要がある」という野球のドメイン知識に基づき、課題を因数分解しました。

Stage 1: まず、球質・コースから「ストライク確率（proba_strike）」を予測するモデルを構築。

Stage 2: そのストライク確率を新たな特徴量として組み込み、最終的な「三振確率」を予測するアンサンブルモデルを構築。

2. 🛠 実運用を想定したデータ処理と特徴量エンジニアリング

精度の追求だけでなく、実務環境（MLOps）での運用に耐えうる堅牢なパイプラインを構築しました。

① ドメイン知識に基づく特徴量作成（Feature Engineering）

物理データの単なる羅列ではなく、実際の野球のセオリーに基づき 9つのオリジナル特徴量 を実装しました。

total_movement（総変化量）

extreme_vertical（極端な縦変化）

effective_speed（リリース位置を考慮した体感速度）

matchup（右投手vs左打者などのプラトーン情報） 等

② 未知データに対するエラーハンドリング（実務力の証明）

推論フェーズ（テストデータ）において、「学習データに存在しなかった未知の球種（例: Split-Finger）」が入力された際、従来の LabelEncoder ではシステムエラーが発生します。本プロジェクトでは、マッピング辞書を作成し、未知のカテゴリを安全に -1（その他） として処理する堅牢な変換パイプラインを実装し、運用時のシステムダウンを未然に防ぐ設計を行いました。

③ リーク（Target Leakage）の厳格な排除

三振を予測する上で、未来の情報（例：その打席での最終的なストライク数）が学習データに混入しないよう、推論時と学習時の特徴量群を厳密に管理・分離しました。

3. 🧠 モデル評価と解釈性（XAI）の担保

現場（監督やコーチ）への説明責任を果たすため、精度評価に加えてモデルの解釈に重きを置きました。

① キャリブレーション（確率の誠実さ）の検証

AIが算出する「三振確率」が、実際のベースレート（約19%）と乖離していないかをキャリブレーションプロットで検証しました。極端な「0か1か」の予測ではなく、現実の確率分布に沿った誠実な（信頼できる）予測ができていることを証明しています。

② SHAPによる局所的解釈（1球ごとの要因分析）

モデルが「なぜその予測に至ったか」を解明するため、SHAP（Waterfall Plot）を用いて以下のケースを解剖しました。

False Positive（騙された球）: AIが「絶対三振する」と高確率を出したにも関わらず、現実ではファウル等にされた球。

False Negative（見逃した球）: AIが「確率が低い」と予測した平凡な球にも関わらず、空振りを奪えた球。

4. 🔬 What-if分析と統計的効果検証（ROIの証明）

「球速を上げればストライクが増える」という非現実的（過酷なトレーニングが必要）なインサイトではなく、現場が明日から実行可能なROIの高いアクションを特定し、その効果を統計的に証明しました。

アプローチ：リリースポイントの微修正によるシミュレーション

仮想データの作成: テストデータに対し、投手の踏み込み幅（release_extension）を現実的に修正可能な範囲（+5cm）前に出した仮想のパラレルデータを作成し、それに伴う体感速度（effective_speed）を再計算。

モデル推論: 現状のデータと仮想データの両方をStage 1（ストライク予測モデル）に入力し、それぞれのストライク確率を算出。

統計的検定: 両確率に対し**「対応のあるt検定（Paired t-test）」**を実施。打者レベルやカウントといった環境ノイズを相殺した純粋な「フォーム修正の効果」を測定しました。

結果: $p < 0.05$ で統計的に有意なストライク確率の向上が確認され、「リリースポイントをわずかに前にするフォーム修正が、三振奪取に向けた極めて費用対効果の高いアクションである」というデータドリブンな証明を行いました。

5. 💡 結論と現場へのアクション提案（BI視点）

発見されたジレンマ：「精度の追求」と「ブラックボックス化」

Stage 2（三振予測）において、新たに作成した「ストライク確率」が強力すぎる変数となり、元々の「球速」や「変化量」がモデル内でマスキングされる現象を確認しました。つまり、三振予測モデル単体では「ストライクを取れそうな球を投げれば三振する」という思考停止に陥ります。

現職のBIスキルを活かしたネクストアクション提案

データ分析の価値は「現場の行動を変えること」です。上記のジレンマを打破し、What-if分析で得られたインサイトをデリバリーするため、もし私が球団のアナリストであれば、Jupyter Notebookの数値をそのまま提出するのではなく、以下のような**「投手向けアクション・ダッシュボード」**を構築・提供します。

📊 投手改善ダッシュボード（提案モックアップ）

KPI: 踏み込み幅（Extension）の達成率、および見逃し・空振りストライク獲得率

アクション目標: 統計的検定で効果が証明された「踏み込み幅+5cm」をベースラインとした、体感速度の目標閾値を可視化。

運用: ブルペンでの投球練習時、トラッキングデータがこの「目標閾値ゾーン」に何％収まっているかをリアルタイムでフィードバックし、フォーム改善の具体的な指標とする。
